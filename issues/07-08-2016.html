Highlights from facebook/react-native

NEW View! - KeyboardAvoidingView

KeyboardAvoidingView is a component we built internally to solve the common problem of views that need to move out of the way of the virtual keyboard.  It can automatically adjust either its position or bottom padding based on the position of the keyboard.

NEW prop! - onContentSizeChange for TextInput

Adds proper support for tracking a TextInput content size by adding a new callback that is called every time the content size changes - including when first rendering the view.

NEW timer function! requestIdleCallback

In this pull request janicduplessis follows up on work by brentvatne in #5052. This addresses the feedback by astreet. This has landed for both iOS and Android in 0.31! requestIdleCallback lets you schedule work for free CPU cycles, see the MDN docs for more details (note: the timeout option is not currently supported).

Change NavigatorSceneConfig for RTL

This is just one of many commits from @MengjueW to add RTL support to react-native, awesome stuff!

Merge `rnpm cli` into react-native

"This is an initial step of rewriting the CLI interface to use `rnpm` one (commander, plugins etc.)." -- in other words, a much more powerful react-native CLI to come!

Show intercepted network information in the inspector

Adds a UI to display network information flows in the inspector tool (Android and iOS both supported)

Convert from React.createClass to ES6 classes

Deprecate:
var MovieCell = React.createClass
Implement:
class MovieCell extends React.Component

Automatic IP detection for iOS for Testing on Device

Implemented automatic IP detection for iOS when testing on device - works by writing the IP address of the host to a file.
